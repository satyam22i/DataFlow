# 📊 Business Analytics Dashboard

A full-stack data analytics application built with **Python + FastAPI** (backend) and **React + Chart.js** (frontend). It ingests raw CSV data, cleans and transforms it, performs business analysis, and presents the results as a beautiful interactive dashboard.

## 🔄 Running the Full Pipeline

Follow these steps **in order** every time you start from scratch.

### Part 1 — Clean the raw data

```bash
python clean_data.py
```

**What it does:**
- Reads `data/raw/customers.csv`, `orders.csv`, `products.csv`
- Removes duplicate customers (keeps most recent `signup_date`)
- Validates and lowercases email addresses, flags malformed ones
- Parses dates in multiple formats (`YYYY-MM-DD`, `DD/MM/YYYY`, `MM-DD-YYYY`)
- Fills missing `amount` values with median grouped by product
- Normalises order `status` to: `completed`, `pending`, `cancelled`, `refunded`
- Adds `order_year_month` column (format: `YYYY-MM`)

**Output files:**
- `data/processed/customers_clean.csv`
- `data/processed/orders_clean.csv`

**Console output** — cleaning report showing rows before/after, nulls removed, duplicates dropped.

---

### Part 2 — Merge & analyse

```bash
python analyze.py
```

**What it does:**
- Left-joins `orders_clean` onto `customers_clean` on `customer_id`
- Left-joins `products.csv` on `product` → `product_name`
- Reports unmatched orders (no customer / no product)

**Generates 4 analysis files in `data/processed/`:**

| File | Contents |
|---|---|
| `monthly_revenue.csv` | Total completed revenue grouped by `order_year_month` |
| `top_customers.csv` | Top 10 customers by total spend + `churned` flag |
| `category_performance.csv` | Revenue, avg order value, order count per category |
| `regional_analysis.csv` | Customers, orders, revenue, avg revenue/customer per region |

**Churn logic:** A customer is flagged `churned = True` if they have had **no completed orders in the last 90 days** (relative to the latest date in the dataset).

---

### Part 3 — Start the backend server

```bash
python -m uvicorn backend.app:app --reload
```

The API will be available at **http://127.0.0.1:8000**

| Endpoint | Description |
|---|---|
| `GET /health` | Health check → `{"status": "ok"}` |
| `GET /api/revenue` | Monthly revenue trend data |
| `GET /api/top-customers` | Top 10 customers with churn status |
| `GET /api/categories` | Category performance breakdown |
| `GET /api/regions` | Regional analysis KPIs |

All endpoints return **JSON**. If a data file is missing, the API returns `HTTP 404` with a descriptive message. CORS is enabled for all origins so the frontend can call the API freely.

Test the API in your browser:
```
http://127.0.0.1:8000/docs        ← Interactive Swagger UI
http://127.0.0.1:8000/api/revenue ← Raw JSON
```

## 📊 Dashboard Features

The dashboard is a single-page React application with the following sections:

### 🔢 KPI Cards (top row)
Five summary cards showing at a glance:
- **Total Revenue** — sum of all completed orders, with month-over-month growth badge
- **Avg Monthly Revenue** — average across all months in the dataset
- **Active Customers** — count of non-churned customers + churn rate
- **Top Category** — highest-earning product category
- **Regions** — number of geographic territories

### 📈 Monthly Revenue Trend
Line chart of completed-order revenue over time.
- **Bonus feature:** Date-range filter using month pickers — narrow the chart to any time window and see live count of months shown.

### 🛒 Revenue by Category (Doughnut chart)
Proportional breakdown of revenue across product categories with an interactive legend.

### 📦 Category Performance (progress bars)
Horizontal progress bars showing each category's share of total revenue, alongside order count and average order value.

### 🗺️ Regional Summary (KPI cards)
One card per region showing: customers, orders, total revenue, and average revenue per customer.

### 🏆 Top 10 Customers Table
Sortable table of highest-spending customers.
- **Bonus feature:** 🔍 **Search box** — filter by customer name or region in real time.
- **Sortable columns** — click any column header to sort ascending/descending.
- **Churn badges** — 🟢 Active or 🔴 Churned shown clearly.
- Medal icons 🥇🥈🥉 for the top 3.

### ⚡ Loading & Error States
- Animated loading screen while data is being fetched
- Friendly error screen with terminal command hint if the backend is unreachable
- **Retry Connection** button to re-attempt without page refresh

### 📱 Responsive Design
- Fully usable on **1280px desktop** (multi-column grid layout)
- Collapses to **single column on mobile (375px)**

## 🐛 Common Issues & Fixes

### Dashboard shows blank / React hook error
**Cause:** `useMemo` or other hooks were called after an early `return` statement.
**Fix:** All hooks must be declared at the top of the component, before any `if (loading) return` or `if (error) return`.

### "Cannot connect to backend" error on dashboard
**Cause:** The FastAPI server isn't running.
**Fix:**
```bash
# Make sure your virtual environment is active
.venv\Scripts\activate          # Windows
source .venv/bin/activate       # Mac/Linux

python -m uvicorn backend.app:app --reload
```

### API returns 404 for data endpoints
**Cause:** The processed CSV files don't exist yet.
**Fix:** Run the pipeline scripts first:
```bash
python clean_data.py
python analyze.py
```

### `ModuleNotFoundError: No module named 'fastapi'`
**Cause:** Virtual environment not activated or dependencies not installed.
**Fix:**
```bash
.venv\Scripts\activate
pip install -r backend/requirements.txt
```

### `npm run dev` fails — missing modules
**Fix:**
```bash
cd frontend
npm install
npm run dev
```

## 📋 Assumptions Made

1. **Currency** — All monetary values are treated as Indian Rupees (₹). No currency conversion is applied.
2. **Churn window** — 90 days is calculated relative to the **maximum `order_date`** in the dataset, not today's date, so the analysis is reproducible regardless of when the script is run.
3. **Status normalisation** — Variants mapped: `done` → `completed`, `complete` → `completed`, `canceled` → `cancelled`, `cancel` → `cancelled`, `return` → `refunded`, `refund` → `refunded`. Any unrecognised value is left as-is with a warning logged.
4. **Duplicate customers** — When two rows share the same `customer_id`, the one with the most recent `signup_date` is kept.
5. **Missing product match** — Orders whose `product` name doesn't match any entry in `products.csv` are kept in the analysis with `category = NaN`. They are excluded from category-level aggregations.
6. **Median fill** — Missing `amount` values are filled with the **median amount of the same product**. If a product has no other orders, the global median is used as fallback.
